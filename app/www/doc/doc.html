Documentation

Introduction

Cette documentation présente l'interface Shiny créée pour l'analyse de données et l'apprentissage de modèles de Machine Learning.
Cette application est destinée à des utilisateurs ayant des connaissances en statistiques et en apprentissage automatique.

Utilisation de l'interface Shiny

Pour utiliser l'interface Shiny, vous devez suivre les étapes suivantes :

  1. Ouvrir l'application Shiny dans votre navigateur web.
  2. Charger le fichier de données à analyser dans l'interface Shiny.
  3. Effectuer le pré-traitement des données et l'analyse exploratoire univariée/bivariée.
  4. Utiliser les modèles de Machine Learning et effectuer des prédictions.

Composantes de l'interface Shiny

L'interface Shiny est constituée de trois composantes :

  1. Le volet dépliant gauche pour charger le fichier de données.
  2. La partie centrale avec un tabpanel contenant trois panels : Dataset, Exploration, Apprentissage de Modèles.

Le volet dépliant gauche

Le volet dépliant gauche permet de charger le fichier de données et d'appliquer des options de chargement. Les options de chargement sont les suivantes :

  1. Séparateur de décimales.
  2. Ignorer les premières lignes.
  3. etc ...

Il est important de noter que la taille maximale du fichier à charger est de 251 mo. Une fois le fichier de données chargé, il apparaîtra dans le panel Dataset.

Dataset panel

Ce panel permet de visualiser les données du dataset chargé ainsi que de choisir les colonnes à étudier et de faire du preprocessing/processing.

Tableau des données

Le tableau des données est affiché contenant le dataset chargé. De plus, chaque colonne peut être filtrée pour afficher seulement les valeurs qui nous intéressent.

Informations sur les colonnes

Un tableau est affiché pour résumer les informations sur chaque colonne : nom, type, nombre de valeurs manquantes...

Choisissez les variables à étudier

Ce bouton permet de sélectionner les variables à étudier dans les onglets suivants. Il est important de ne sélectionner que les variables qui ont du sens pour l'analyse exploratoire et l'apprentissage de modèles.

Opération sur une colonne

Trois listes déroulantes et un bouton "Valider la catégorie de la variable" ou "Valider la saisie" sont présents:

  1. Ces listes déroulantes servent à modifier une variable parmi les variables à étudier (exemple : supprimer les outliers pour une variable quantitative.)
  2. Une fois la selection faite, il faudra impérativement cliquer sur le bouton de validation (tout à droite des listes déroulantes) pour valider l'opération.
  3. Si le type d'opération n'est pas compatible avec la variable sélectionnée (liste déroulante :"Choisissez une variable"). L'opération ne sera pas effectuée même après avoir appuyé sur le bouton de validation.

Faire Preprocessing

Ce bouton permet de faire du preprocessing sur toutes les colonnes avec suppression de valeurs manquantes avec imputation (moyenne/mode selon le type de variable).
Et de "caster" le type de variable selon la catégorie (quantitative discrète, quantitative continue, qualitative ordinale, qualitative nominale).
Il est important de faire le preprocessing avant de passer à l'analyse exploratoire et l'apprentissage de modèles.

Exploration panel

Ce panel contient l'analyse exploratoire univariée/ bivariée.
L'utilisation est intuitive, il faudra faire d'abord le processing des colonnes avant de rentrer dans ce panel.
Les graphiques proposés dans les listes déroulantes sont conditionnées selon la/les catégories de/des variable(s).

Apprentissage de Modèles panel

Une fois l'analyse exploratoire faite, ce panel concerne l'apprentissage de modèles de classification de ML. Il contient de nombreuses options (choix de la proportion du test set, choix du seuil d'acceptation...).

Trois modèles ont été implémentés (Regression Logistique, Arbre de Décision CART, Arbre de Décision CHAID):

  1. La sélection de modèle est dynamique, c'est-à-dire que certains modèles pour un certain dataset ne seront pas disponibles car les critères sur les colonnes ne sont pas respectés.
    Exemple : si on a une feature qualitative, on ne peut pas entraîner un modèle de Regression Logistique.
    Donc, à l'affichage, il n'y aura pas de modèle de Regression Logistique disponible.
    Il faudra dummifier la variable au préalable via le Dataset panel et sélectionner que des variables quantitatives.
  2. Il y a aussi une liste déroulante "Choisir le type d'équilibrage" avec un curseur "Prop. classe minoritaire" et un bouton "Valider Over/Under sampling".
    Ces boutons sont affichés dynamiquement lorsque la variable à prédire pour un modèle de ML est qualitative à 2 MODALITÉS.
    Ils servent donc à rééquilibrer le jeu de données en réglant la proportion de lignes de la classe minoritaire qui ne peut pas être plus de 0.5 (50%).
  3. Une fois toutes les options choisies, il faudra cliquer sur "Valider les options" pour lancer le calcul du modèle de ML choisi.
  4. Si il n'y a pas de variable qualitative dans le dataset sélectionné via Dataset panel:
    Le bouton "Valider les options" et d'autres boutons d'options du modèle de ML ne sont pas affichés.
    Car ce sont des modèles de classification et non de régression. L'aspect dynamique du panel a été réfléchi de manière à éviter les erreurs de saisie utilisateur.
